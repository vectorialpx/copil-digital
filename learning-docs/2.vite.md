# Despre vite.dev

In proiectele JavaScript este foarte important sa folosesti pachete deja facute, sa nu reinventezi roata in fiecare proiect, pentru a castiga timp de dezvoltare.

## Despre package.json

Unu din modurile de a importa pachete deja existente este sa folosim `npm`, care vine la pachet cu NodeJs. In acest proiect o sa folosim NodeJs 22, care are `npm` 11.4

## Transpile

JavaScript a avansat foarte mult si, acum exista o multime de moduri sa incepi un proiect front-end. Dar, ce inseamna sa incepi?

Majoritatea browserelor inteleg JavaScript ES5 (EcmaScript 5), care este o versiune mai veche de JavaScript. De ce inteleg o versiune veche si nu cea mai noua?... Aici discutia e lunga, pur si simplu fiecare browser este si el un software, care are propria implementare - unele inteleg ES6 direct, altele nu. Asa ca, pentru a fi safe, pentru a putea scrie cod inteles de toata lumea, cel mai sigur este sa expunem cea mai mica versiune.

Partea buna este ca nu trebuie sa scrime cod in ES5, putem sa-l scriem in ES6 si apoi sa-l trecem printr-un proces de transplile, adica de traducere.

Exista anumite pachete care ne pot ajuta cu acest transpile, cum este `webpack`, dar lucrurile se complica destul de repede, pentru ca trebuie configurat foarte bine ca sa ajungi la un rezultat bun si, conform cu nevoile tale.

## Vite

Un mod foarte simplu de a incepe un proiect este sa folosim Vite, un pachet (sau utilitar) care ne ofera un proiect deja functional, dintr-o simpla linie de comanda.

### Cum se foloseste Vite

```bash
npm create vite@latest client -- --template react
```

Prima intrebare este `Install with npm and start now?` si daca spunem `y`es, vom si avea proiectul gata pregatit pe adresa `http://localhost:5173/`

Momentan, in acest commit doar am instalat Vite, pe care il poti porni folosind:

```bash
npm run dev
```
